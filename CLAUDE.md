# 개발 환경
- OS: Windows 11 Pro
- 언어: TypeScript
- 프레임워크: Next.js 16, React 19

# 언어 및 커뮤니케이션 규칙
- 기본 응답 언어: 한국어
- 코드 주석: 한국어로 작성
- 커밋 메시지: 한국어로 작성
- 문서화: 한국어로 작성 (IMPORTANT)
- 변수명/함수명: 영어 (코드 표준 준수)

# 코딩 스타일
- 들여쓰기: 2칸
- 네이밍: camelCase, PascalCase (컴포넌트)

# 기술 스택
- CSS: Tailwind CSS
- UI: shadcn/ui
- 상태관리: Zustand (향후 추가 예정)
- 폼: React Hook Form + Zod

# 추가 사항
- any 타입 사용 금지
- 컴포넌트 분리 및 재사용
- 반응형 필수

# 모든 프로젝트에서 README.md 파일 작성시 지켜야 할 사항 (IMPORTANT)

1. 프로젝트의 환경(운영체제, 언어, 폴더구조 등) 이 새로 만들어지거나, 변경될 때마다 README.md 파일을 업데이트 할 것.
2. 한글로 작성할 것
3. 아래의 구조로 작성

  * 서비스 개요
  * 서비스 실행 방법
  * 폴더 구조와 각 폴더의 역할
  * 백앤드, 프론트에 사용된 버전 및 소스 경로
  * DB(oracle, mysql)의 레벨별 properties 설정
  * 서비스 레벨 별 properties 파일 위치 변수(dev, test, prod)
  * (로깅 설정이 있을 경우) 로깅 구조 및 설정파일 위치
